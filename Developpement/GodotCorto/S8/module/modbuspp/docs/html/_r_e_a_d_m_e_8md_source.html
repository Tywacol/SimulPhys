1 # MODBUS++ [(MODBSPP)](https://github.com/fanzhe98/modbuspp.git) V0.1
2 MODBUS++ (MODBUSPP) is an open source c++ class/ library making an encapsulation of [Modbus](https://en.wikipedia.org/wiki/Modbus) TCP Protocol published by Modicon (Now Schneider Electirc).
3 
4 MODBUS++ is based on Object-Oriented Programming. While it keeps the efficiency of C++ Code , it provides a higher level of abstraction than other C Modbus Library. Generally, it is easier for programmers to use in their development requiring MODBUS TCP Protocol. Compared to other known modbus library, such as libmodbus, MODBUS++ is providing a more OOP friendly syntaxes.
5 
6 The code has dependencies on libary on Linux for TCP/IP, if you want to use this in Windows, please check out winsock2 and rewrite portion of code of socket to be compatible with Windows operating system.

29 ## 2.2 Tutorials
30 Let's break the code down, before you read this, BE SURE to have a reference to [example.cpp](https://github.com/fanzhe98/modbuspp/blob/master/example.cpp). The example shows basic usages of how to create a modbus client connecting to a modbus server and perform modbus operations in your program.
31 ### Include the header
32 
33 To start with, be sure to include the modbus.h header in your program
34 > #include "modbus.h"
35 
36 This line includes the header modbus.h from the library, this will tell the compiler to look for functions and variables in the related file.
37 
38 ### Create and connects a mobus server
39 Before performing any modbus operations, a connection needed to be setup between the modbus client(your program) and a modbus server(could be a controller).
40 
41 > modbus mb = modbus("127.0.0.1", 502);
42 > mb.modbus_set_slave_id(1);
43 > mb.modbus_connect();
44 
45 These lines creates local modbus client with target IP address at 127.0.0.1 and port at 502, a server with id 1.
46 
47 ### Performe Mobus Operations
48 Modbus usually have 2 types of operations, reading and writing. Reading operations include reading a coil, reading input bits, reading holding registers, and reading input registers. Writing Operations include writing sigle coil, wirting single register, writing multiple coil, and writing multiple registers. Be sure to create a connection before performing any operations. Modbus is relatively low-level protocols. It is useful to have strong computer structure knowledge to understand it.
49 
50 1. The following line show how to read a coil (function 0x01). These lines read a bit from the coil at address 0.
51 > bool read_coil;
52 > mb.modbus_read_coils(0, 1, &read_coil);
53 
54 2. The following lines show how to read input bits(discrete input) (function 0x02). These lines read a bit from discrete inputs at address 0.
55 > bool read_bits;
56 > mb.modbus_read_input_bits(0, 1, &read_bits);
57 
58 3. The follwing lines show how to read holding registers (function 0x03). These lines read a word(16 bits) from holding registers starting at address 0.
59 > uint16_t read_holding_regs[1];
60 > mb.modbus_read_holding_registers(0, 1, read_holding_regs);
61 
62 4. The following lines show how to Read input registers (function 0x04). These lines read a word(16 bits) from input registers starting at address 0.
63 > uint16_t read_input_regs[1];
64 > mb.modbus_read_input_registers(0, 1, read_input_regs);
65 
66 5. The following lines show how to write single coil (function 0x05). These lines write a bit to the coil at address 0.
67 > mb.modbus_write_coil(0, true);
68 
69 6. The following lines show how to write single register (function 0x06). These lines write a word(16 bits) to the register at address 0.
70 > mb.modbus_write_register(0, 123);
71 
72 7. The following lines show how to write multiple coils (function 0x0F). These lines write multiple bits to the coil starting from address 0, for a length of 4.
73 > bool write_cols[4] = {true, true, true, true};
74 > mb.modbus_write_coils(0,4,write_cols);
75 
76 8. The following lines show how to write multiple regs (function 0x10). These lines write multiple words to the register starting from address 0, for a length of 4.
77 > uint16_t write_regs[4] = {123, 123, 123};
78 > mb.modbus_write_registers(0, 4, write_regs);
